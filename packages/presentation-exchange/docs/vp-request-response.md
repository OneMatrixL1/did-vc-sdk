# VP Request / Response

All sample data in this document is **real** — generated by running actual ZKP circuits (UltraHonk via bb.js). No mocks.

Regenerate: `cd packages/presentation-exchange && npx vitest run __tests__/generate-sample-data.test.ts`

---

## Flow

```
Verifier                          Holder                           Verifier
────────                          ──────                           ────────
1. Build VPRequest         ──▶
                                  2. Match credentials
                                  3. Resolve VP (ZKP proofs)
                           ◀──    4. Send VP Response
                                                                   5. Verify structure
                                                                   6. Verify ZKP chain
```

## 1. VP Request

Verifier asks: "Show me fullName and gender from your CCCD."

```json
{
  "type": ["VerifiablePresentationRequest"],
  "id": "kyc-001",
  "version": "2.0",
  "name": "Bank KYC Verification",
  "nonce": "nonce-abc123",
  "verifier": "did:web:bank.vn",
  "verifierName": "Vietnam International Bank",
  "verifierUrl": "https://bank.vn",
  "rules": {
    "type": "DocumentRequest",
    "docRequestID": "cccd",
    "docType": ["CCCDCredential"],
    "schemaType": "ICAO9303SOD",
    "name": "National ID (CCCD)",
    "conditions": [
      { "type": "DocumentCondition", "conditionID": "fullName", "field": "fullName", "operator": "disclose" },
      { "type": "DocumentCondition", "conditionID": "gender", "field": "gender", "operator": "disclose" }
    ]
  }
}
```

### Builder code

```ts
const request = new VPRequestBuilder('kyc-001', 'nonce-abc123')
  .setName('Bank KYC Verification')
  .setVerifier({ id: 'did:web:bank.vn', name: 'Vietnam International Bank', url: 'https://bank.vn' })
  .addDocumentRequest(
    new DocumentRequestBuilder('cccd', 'CCCDCredential')
      .setSchemaType('ICAO9303SOD')
      .setName('National ID (CCCD)')
      .disclose({ field: 'fullName' })
      .disclose({ field: 'gender' }),
  )
  .build();
```

## 2. VP Request with Predicates

Multi-document request: student CCCD + parent CCCD, with date range and cross-document equality.

```json
{
  "type": ["VerifiablePresentationRequest"],
  "id": "enroll-001",
  "version": "2.0",
  "name": "School Enrollment Check",
  "nonce": "nonce-xyz789",
  "verifier": "did:web:school.edu.vn",
  "verifierName": "HCMC University",
  "verifierUrl": "https://school.edu.vn",
  "rules": {
    "type": "Logical",
    "operator": "AND",
    "values": [
      {
        "type": "DocumentRequest",
        "docRequestID": "student-cccd",
        "docType": ["CCCDCredential"],
        "schemaType": "ICAO9303SOD",
        "name": "Student National ID",
        "conditions": [
          { "type": "DocumentCondition", "conditionID": "fullName", "field": "fullName", "operator": "disclose" },
          { "type": "DocumentCondition", "conditionID": "dateOfBirth", "operator": "inRange", "field": "dateOfBirth",
            "params": { "gte": "19900101", "lte": "20061231" } },
          { "type": "DocumentCondition", "conditionID": "dob-gte", "operator": "greaterThanOrEqual", "field": "dateOfBirth",
            "params": { "value": "19900101" } }
        ]
      },
      {
        "type": "DocumentRequest",
        "docRequestID": "parent-cccd",
        "docType": ["CCCDCredential"],
        "schemaType": "ICAO9303SOD",
        "name": "Parent National ID",
        "conditions": [
          { "type": "DocumentCondition", "conditionID": "fullName", "field": "fullName", "operator": "disclose" },
          { "type": "DocumentCondition", "conditionID": "parent-match", "operator": "equals", "field": "fullName",
            "params": { "ref": "student-cccd.fatherName" } }
        ]
      }
    ]
  }
}
```

### Builder code

```ts
const request = new VPRequestBuilder('enroll-001', 'nonce-xyz789')
  .setName('School Enrollment Check')
  .setVerifier({ id: 'did:web:school.edu.vn', name: 'HCMC University', url: 'https://school.edu.vn' })
  .addDocumentRequest(
    new DocumentRequestBuilder('student-cccd', 'CCCDCredential')
      .setSchemaType('ICAO9303SOD')
      .disclose({ field: 'fullName' })
      .inRange({ field: 'dateOfBirth', gte: '19900101', lte: '20061231' })
      .greaterThanOrEqual({ field: 'dateOfBirth', value: '19900101', id: 'dob-gte' }),
  )
  .addDocumentRequest(
    new DocumentRequestBuilder('parent-cccd', 'CCCDCredential')
      .setSchemaType('ICAO9303SOD')
      .disclose({ field: 'fullName' })
      .equals({ field: 'fullName', ref: 'student-cccd.fatherName', id: 'parent-match' }),
  )
  .build();
```

### Condition operators

All methods accept an object. `id` is optional — defaults to `field`.

| Method | Params | Circuit |
|--------|--------|---------|
| `.disclose({ field })` | — | `dg13-field-reveal` |
| `.greaterThan({ field, value })` | `{value}` | `date-greaterthan` |
| `.lessThan({ field, value })` | `{value}` | `date-lessthan` |
| `.greaterThanOrEqual({ field, value })` | `{value}` | `date-greaterthanorequal` |
| `.lessThanOrEqual({ field, value })` | `{value}` | `date-lessthanorequal` |
| `.inRange({ field, gte, lte })` | `{gte, lte}` | `date-inrange` |
| `.equals({ field, ref })` | `{ref}` | `field-equals` |
| `.equals({ field, value })` | `{value}` | `field-equals` |

## 5. ICAO9303ZKPProofBundle

The credential proof is a flat array of `ZKPProofEntry`. Each entry is self-describing via `circuitId`. Proof values truncated — full data in `docs/samples/vp-response-full.json`.

```
proofs: [
  { circuitId: "sod-verify",       ... }  ─┐
  { circuitId: "dg-map",           ... }   │ binding chain
  { circuitId: "dg13-merklelize",  ... }  ─┘
  { circuitId: "dg13-field-reveal", conditionID: "fullName", ... }  ← reveals
  { circuitId: "dg13-field-reveal", conditionID: "gender",   ... }
]
```

### Binding chain

```
sod-verify.econtent_binding ═══▶ dg-map.input.econtent_binding
dg-map.dg_binding           ═══▶ dg13-merklelize.binding
dg13.commitment             ═══▶ field-reveal.input.commitment (all reveals)
```

### Circuit outputs (named)

| Circuit | Outputs |
|---------|---------|
| `sod-verify` | `econtent_binding` |
| `dg-map` | `dg_binding` |
| `dg13-merklelize` | `binding`, `identity`, `commitment` |
| `dg13-field-reveal` | `length`, `data_0`, `data_1`, `data_2`, `data_3` |

### ZKPProofEntry structure

Every entry in `proofs[]` has this shape:

```ts
{
  circuitId: string,                          // which circuit
  proofValue: string,                         // ZKP proof (base64)
  publicInputs: Record<string, unknown>,      // verifier sees
  publicOutputs: Record<string, unknown>,     // named circuit outputs
  conditionID?: string,                       // links to request condition (absent for chain proofs)
}
```

The verifier resolves `field`, `operator`, `params` from the request via `conditionID` lookup — no duplication in the proof entry.

## Verification (two layers)

### Layer 1: zkp-provider (pure math)

Verify every proof independently. No domain knowledge needed.

```ts
for (const entry of bundle.proofs) {
  const valid = await zkpProvider.verify({
    circuitId: entry.circuitId,
    proofValue: entry.proofValue,
    publicInputs: entry.publicInputs,
    publicOutputs: entry.publicOutputs,
  });
}
```

### Layer 2: presentation-exchange (binding chain)

Check that proofs are linked correctly.

1. **Structural** — VP fields match request (holder, verifier, nonce, submission entries)
2. **sod-verify → dg-map** — `econtent_binding` input matches `econtent_binding` output
3. **dg-map → dg13** — `dg13.binding == dg-map.dg_binding` (same DG13 hash)
4. **dg13 → reveals** — all `field-reveal.input.commitment == dg13.commitment`
5. **dg13 → predicates** — same commitment binding
6. **Completeness** — all required conditions have matching proof entries

If any link breaks, the chain is invalid.

### Decoded values

`credentialSubject` is intentionally empty — no schema exists for a partially-disclosed credential.
Revealed field values are decoded from `publicOutputs` (`length` + `data_0..data_3`) during verification:

```ts
const result = await verifyVPResponse(request, vp, { zkpProvider });
// result.disclosedFields.fullName → "Nguyen Thi B"
// result.disclosedFields.gender   → "Nu"
```
